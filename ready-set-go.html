Ready, set, Go!|

Hi! This is my first blog post. I've been learning about the Go programming language lately and thought it was an interesting topic to start with.

Golang (as known as Go) is a free and open source programming language being developed by Google development since 2007. In 2012, the language reached the 1.0 milestone.

The best of both worlds?

Go is being designed and implemented by brilliant minds like Ken Thompson and Rob Pike, among others, in an attempt to fill the gaps where Python, Java and C++ have room for improvement.

It aims to mix some of the best features from these and other modern programming languages, with a very nice balance between power and expresiveness: clear and concise syntax but still statically typed and compiled directly to machine code, high-performance and efficient code, garbage collection, native concurrency (no OS threads), etc, making it ideal to write small, focused APIs.

Simplicity and integrity are key parts of the language design. For example, you don't have to necessarily declare the type of a variable, type inference take cares of that. Variables have a predefined zero value (0 for numeric types, empty for strings, nil for pointers, etc) so you also don't have to care about initializing them, making it less error prone.

Some kind of object-orientation with functional programming features are given, but no object hierarchy or classes.
